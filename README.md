## Ответы на контрольные вопросы (вопросы для интервью)
**1. _В чем разница между map ( ) и forEach ( )? Когда использовать каждый?_**
  > `forEach()` используется для простого перебора, когда нужно выполнить действие для каждого элемента (например, отрисовка карточек в Задании 1). Он ничего не возвращает. `map()` используется для трансформации данных и всегда возвращает новый массив той же длины. В Задании 3 мы использовали `map()` для создания нового массива объектов с дополнительным полем `label`, не изменяя исходный массив `products`.

**2. _Что возвращает filter ( )? Изменяет ли он исходный массив?_**
  > `filter()` возвращает новый массив, содержащий только те элементы, которые соответствуют условию (условие вернуло `true`). Он не изменяет исходный массив. Это позволило нам реализовать поиск по названию и категории (Задание 2 и 3), сохраняя при этом полный список товаров в переменной `products` для повторных поисков.

**3. _Что такое JSON? Чем JSON-строка отличается от JavaScript-объекта?_**
  > `JSON` (`JavaScript Object Notation`) — это текстовый формат данных. Разница в том, что объект — это структура данных в памяти, а `JSON` — это строго отформатированная строка. В `JSON` ключи обязательно пишутся в двойных кавычках. В Задании 5 мы использовали `JSON.stringify()`, чтобы превратить массив cart в строку для сохранения в `localStorage`.

**4. _Что произойдет, если передать в JSON.parse ( ) невалидную строку?_**
  > Произойдет ошибка синтаксиса (`SyntaxError`), и скрипт перестанет работать. При загрузке данных из `localStorage` (Задание 5) мы сначала проверяем наличие данных через if (saved), чтобы избежать передачи пустого значения или некорректной строки в `JSON.parse()`.

**5. _Что такое деструктуризация объекта? Приведите пример из своего кода._**
  > Это способ быстрого извлечения свойств объекта в переменные. Хотя в теории чаще используется прямой доступ через точку (например, `p.name`), в коде при создании карточек мы можем использовать деструктуризацию: `const { name, price } = product`; вместо `product.name` и `product.price`. Это делает код чище.

**6. _Что делает оператор spread (…) при копировании объекта?_**
  > Он «разворачивает» свойства существующего объекта в новый объект. В практической работе использовали этот оператор в цепочке методов Задания 3.  В частности, `...p` позволяет создать копию товара и добавить к ней поле `label`, не модифицируя оригинальный объект в массиве `products`.

**7. _Почему нельзя сравнивать объекты через ===? Что выведет: { } === { }?_**
 > Объекты сравниваются по ссылке (адресу в памяти), а не по содержимому. Выражение выведет `false`, так как это два разных объекта в разных ячейках памяти. По этой же причине в Задании 4 для проверки наличия товара в корзине мы сравниваем не объекты целиком, а их уникальные `id`.

**8. _Что такое метод reduce ( )? Как вы использовали его для подсчета суммы?_**
  > Это метод-агрегатор, который перебирает массив и накапливает результат в одной переменной (аккумуляторе). В Задании 4 и 6 использовали его для вычисления итоговой суммы корзины и средней цены. В частности, формула: `prices.reduce((sum, price) => sum + price, 0)`, где 0 — начальное значение суммы.
